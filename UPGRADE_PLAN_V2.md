# 浅草缘签 · 赛博神社 2.0 升级蓝图

> 在 `UPGRADE_PLAN.md`（v1）基础上整合、深化、重排优先级  
> 目标：从「能抽签的网页」→「有完整参拜叙事、视觉冲击与回访动力的数字浅草寺」

---

## 0. 现状诊断（基于当前代码）

| 维度 | 现状 | 升级方向 |
|------|------|----------|
| 核心流程 | 首页选模式 → 九宫格 → 随机 1/100 | 多阶段仪式流 + 可选摇签筒 |
| 九宫格语义 | 9 个抽屉编号不同，但点击均 `drawRandomOmikuji()` | 要么真随机，要么「摇签筒定号 + 九宫格选格」叙事一致 |
| 手势 | MediaPipe 悬停/捏合/换批，体验已领先 | 扩展合十、摇筒、解厄仪式手势 |
| 数据 | 100 签 JSON 完整（签诗/解读/words/正反面图） | 打标签、加权抽签、图鉴、合盘 |
| 存储 | localStorage 5 条 `{id, drawnAt}` | IndexedDB + 统计/成就/收藏 |
| 架构 | 单文件 `main.ts` ~740 行 | 模块化，必要时再上 Svelte |
| 视觉 | 和纸色 + 宋体 + 基础卡片 | 粒子/卷轴/季节/夜景/分享卡 |

**设计原则（延续 v1，并强化）：**

1. **慢交互** — 每一步都有反馈，不跳过仪式  
2. **文化可信** — 浅草寺百签规则、凶签系树、未吉解释要准确  
3. **温柔体验** — 凶签有「解厄」路径，大吉不浮夸  
4. **纯前端优先** — 社交榜/广场等需后端的功能放最后

## 1. 产品愿景：三层体验

```
┌─────────────────────────────────────────────────────────┐
│  L3 · 传播层   分享卡 / 合盘 / 每日签榜 / URL permalink   │
├─────────────────────────────────────────────────────────┤
│  L2 · 留存层   图鉴 / 成就 / 统计 / 收藏 / 每日一签       │
├─────────────────────────────────────────────────────────┤
│  L1 · 仪式层   参拜之旅 / 摇签筒 / 揭签 / 解厄 / 音效氛围  │  ← 优先做
└─────────────────────────────────────────────────────────┘
```

用户打开页面的第一感受应是：**走进一座会呼吸的浅草寺**，而不是打开一个表单。

## 2. 核心差异化：「参拜之旅」叙事流（新 · 最高优先级）

将现有「选模式 → 九宫格 → 结果」升级为**可跳过的完整参拜**，默认推荐走全流程：

```
[山门] 雷门视差 + 简介
    ↓
[净手] 点击/手势模拟洗手（水纹粒子）
    ↓
[投币] 拖入 5 円 至 赛博功德箱（可选跳过）
    ↓
[祈愿] 输入一句心愿（可选，影响签种权重，见 §4）
    ↓
[抽签] 摇签筒 或 九宫格（二选一或串联）
    ↓
[揭签] 卷轴展开 + 墨迹渗开 + 吉凶书法字
    ↓
[解签] 折叠解读 + 正反面签图
    ↓
[善后] 大吉→「带走签文」 / 凶→「系在签树上」解厄动画
    ↓
[离寺] 分享 / 收藏 / 再看统计
```

**技术要点：**

- 用 `AppState` 状态机管理阶段：`gate | purify | offer | wish | draw | reveal | resolve | exit`
- 每阶段独立组件 + 统一 `transition` 层（CSS + 可选 View Transitions API）
- 进度指示：顶部「参道」圆点或鸟居图标条，可点击回看已完成步骤

**炫酷点：** 首次进入自动播放 3s 山门入场；再次进入记住进度偏好（localStorage）。

## 3. 感官系统（合并 v1 阶段一 + 增强）

### 3.1 视觉氛围矩阵

| 模块 | 描述 | 实现 | 工期 |
|------|------|------|------|
| 和纸/金箔底 | 多层 blend 纹理，不抢内容 | CSS `background-blend-mode` + 低 opacity PNG | 0.5d |
| 樱花/红叶/雪粒子 | 按季节切换粒子形态与颜色 | 单 Canvas `ParticleEngine` 类，参数化 | 1d |
| 昼夜主题 | 6–18 暖金 / 18–6 月光蓝 + 灯笼光晕 | `:root` CSS 变量 + `prefers-color-scheme` | 0.5d |
| **深夜神社模式** | 手动切换：萤火虫、灯笼摇曳、更低 BGM | 独立 theme token 集 | 1d |
| 香炉烟雾 | 抽签/加载时角落升烟 | Canvas Bezier 粒子，30 粒子内 | 0.5d |
| **水墨晕染背景** | 鼠标/手指划过留墨迹，渐淡 | 离屏 Canvas + `globalCompositeOperation: multiply` | 1d |
| 雷门视差 | 山门三层 SVG/PNG 随陀螺仪或鼠标偏移 | `transform: translate3d` + DeviceOrientation | 1d |

### 3.2 音效系统（Howler 或 Web Audio）

| 场景 | 音效 | 备注 |
|------|------|------|
| 环境 | 鸟鸣、远钟、风铃循环 | 10s loop，`GainNode` 默认 0.15 |
| 净手 | 水流 | 短音 |
| 投币 | 铜钱落箱 | 短音 |
| 摇筒 | 竹签碰撞 escalating | 随摇动幅度增益 |
| 揭签 | 纸展开沙沙声 | |
| 凶签解厄 | 低钟 + 风过 | 安抚向，非惊悚 |
| 控制 | 角落 🔇 + 音量条 | `localStorage` 持久化 |

**注意：** 首次交互后再 `AudioContext.resume()`，遵守浏览器自动播放策略。

### 3.3 揭签演出（结果页质变）

1. **卷轴展开**：`clip-path` 或双半屏 `scaleX` 从中心展开  
2. **竖排签诗**：`writing-mode: vertical-rl`，从右向左逐列 `opacity` + `translateY`  
3. **墨迹渗开**：每行文字 `mask-image` 动画模拟洇染（或 Canvas 逐字）  
4. **吉凶大字**：`大吉` / `凶` 等用书法体 fallback + `text-shadow` 光晕，背景色随 grade 渐变  
5. **签图翻转**：正面 → 点击翻牌看背面（3D `rotateY`）  
6. **详细解读折叠**：words 各项默认折叠，展开带 accordion 动画

## 4. 交互深化（合并 v1 阶段二 + 新玩法）

### 4.1 摇签筒（核心差异化）

- Canvas 绘制桶形签筒 + N 根签棒（竖线 + 顶部色块）  
- **输入**：鼠标/触摸左右拖、手机 `devicemotion` 摇晃强度  
- **物理**：弹簧 + 阻尼正弦，`shakePower` 累积阈值后一根签抛物线飞出  
- **落签**：贝塞尔落点 → 显示编号 → 引导「前往选屉」或直接揭晓  
- 与九宫格关系：**摇出编号 mod 9 高亮对应屉**，再捏合/点击确认（叙事一致）

### 4.2 九宫格升级

- 抽屉 **stagger 弹入**（`animation-delay: index * 80ms`）  
- 选中屉 **拉开动画**（`transform: translateY` + 内阴影）  
- 换批：签棒洗牌音效 + 数字重排（仅视觉，不改变即将抽中的签）  
- **修复语义**：若不做摇筒，九宫格应标注「择屉问缘，签文由神明随机赐予」避免用户误以为屉号=签号

### 4.3 手势 2.0

| 手势 | 检测逻辑 | 效果 |
|------|----------|------|
| 悬停选屉 | 现有 indexTip 映射 grid | 保留 |
| 捏合确认 | thumb-index < 0.4 palm，保持 2s | 保留 |
| 中指捏合换批 | 保留 | 保留 |
| **双手合十** | 双手手腕距离 < 阈值，指尖向上 | 进入祈愿/抽签神圣态，UI 变暗聚焦 |
| **摇筒** | 握拳 + 手腕高频率位移 | 驱动签筒 shakePower |
| **鞠躬** | 手 Y 轴持续下移 1s | 解厄仪式触发 |
| **摊掌** | 五指张开，掌心朝镜头 | 签诗 AR 浮于掌心（2D overlay 即可） |
| 引导层 | 骨架线 + 当前识别手势图标 | 降低学习成本 |

### 4.4 签种与加权抽签（需数据预处理）

在 `contents.json` 每条增加扩展字段（脚本批量生成）：

```ts
type OmikujiMeta = {
  tags: ("love" | "career" | "health" | "travel" | "wealth")[];
  tone: "auspicious" | "neutral" | "caution"; // 用于 BGM/配色
  shortLine?: string; // 今日一言用的一句摘录
};
```

| 模式 | 逻辑 |
|------|------|
| 综合运势 | 均匀随机 1–100 |
| 爱情/事业/… | 在 `tags` 含对应项的签中加权随机 |
| 今日一言 | 抽 `shortLine`，UI 极简一页 |
| 指定数字签 | 输入 1–100，直接展示（浅草「指定抽签」文化） |
| 心愿加权 | 用户输入关键词 → 简单关键词表匹配 tags 加权（纯前端，无 AI 也可） |

### 4.5 凶签解厄仪式（情感设计 · 强烈推荐）

当 `grade` 为凶/大凶时，结果页不直接结束：

1. 文案温柔提示：「凶签留在寺中，厄运随之化解」  
2. 用户将签文拖向虚拟 **签树/铁丝网**（浅草实景还原）  
3. 系签动画 + 风铃一声 + 粒子上升  
4. 记录 `purified: true` 到历史（成就「化解厄运」）

## 5. 数据与留存（合并 v1 阶段三）

### 5.1 IndexedDB 模型

```ts
// store: records
interface DrawRecord {
  id: string;           // uuid
  fortuneId: number;
  grade: string;
  signType: "general" | "love" | "career" | "daily" | "specified";
  wish?: string;
  method: "touch" | "gesture" | "shake";
  purified?: boolean;
  drawnAt: string;      // ISO
}

// store: collections — fortuneId[]
// store: achievements — achievementId + unlockedAt
// store: prefs — sound, theme, seasonOverride, tutorialDone
// store: daily — { date: "2026-05-23", fortuneId?: number }  // 每日一签
```

库推荐：`idb` 或 `idb-keyval`（轻量）。

### 5.2 统计看板

- 吉凶分布：**环形图**（Chart.js 或纯 SVG）  
- 近 30 次趋势：折线（按 tone 打分 -2~+2）  
- **吉凶日历**：GitHub 贡献图风格，颜色=grade  
- 时段热力：早/午/晚抽签运势对比  
- 签种完成度：爱情签抽过多少种不同签  

### 5.3 图鉴与成就

| 成就示例 | 条件 |
|----------|------|
| 初参 | 完成首次参拜之旅全流程 |
| 百签收藏家 | 图鉴解锁 100/100 |
| 连吉七日 | 连续 7 天抽到吉以上（娱乐向） |
| 解厄者 | 系签 10 次 |
| 手势大师 | 手势模式抽签 20 次 |
| 深夜参拜 | 深夜神社模式下抽签 |

图鉴页：100 格网格，未抽中灰色剪影，已抽显示缩略 grade + 点击查看详情。

### 5.4 每日一签

- 每天首次抽签特殊边框 + 不可重复抽（可设置「再抽需确认」打破限制）  
- 推送：PWA 阶段再做 Notification

### 7.2 迁移策略

1. **Phase A**：从 `main.ts` 抽出 `services/` + `components/`，保持 Vanilla TS  
2. **Phase B**：引入轻量路由（`?page=draw`）  
3. **Phase C**：若单文件状态 > 5 个互相关联页面，再评估 **Svelte**（v1 结论仍成立）  

### 7.3 PWA 与性能

- `vite-plugin-pwa`：缓存 JSON + 图片 + MediaPipe CDN fallback  
- 图片：首屏 WebP，懒加载保留  
- 手势：`querySelector` 结果缓存，避免每帧查询  
- 粒子：移动端粒子数减半；可选 OffscreenCanvas

## 13. 风险与边界

| 项 | 说明 |
