## 1. 总体愿景

**目标：** 将浅草缘签打造成一个集**文化仪式感、视觉效果、交互沉浸感**于一体的赛博签筒体验，让用户打开页面就像走进了一座数字化的浅草寺。

**核心理念：**
- **慢交互** — 不追求快，追求仪式感。每一步都是体验。
- **文化质感** — 每个视觉细节都致敬日式寺庙美学。
- **温柔反馈** — 抽到凶签不焦虑，抽到大吉不浮躁。

## 2. 阶段一：感官升级 · 沉浸式体验

### 2.1 环境氛围渲染 🎨

| 功能 | 描述 | 技术方案 |
|------|------|----------|
| **动态背景** | 叠加半透明的水墨纹理 / 和纸纹理 / 金箔质感，不抢内容又有层次 | CSS `background-blend-mode` + gradient + 低透明度 `<canvas>` 粒子 |
| **樱花飘落动画** | 背景中飘落樱花花瓣，鼠标/手指滑过时花瓣散开 | Canvas 粒子系统（轻量，50-80 粒子） |
| **光影变化** | 根据时间自动切换暖/冷色调——傍晚暖金色，深夜月光蓝 | CSS 变量在 `:root` 上动态切换 + `matchMedia('prefers-color-scheme')` 可选 |
| **季节主题** | 春季樱花、夏季风铃/团扇、秋季红叶、冬季初雪，自动匹配月份 | 按月切换素材包 + CSS 变量集 |
| **香炉 / 焚香动画** | 页面加载/抽签时，角落浮现一缕香烟升腾的动画 | Canvas 烟雾粒子模拟（Bezier 曲线路径） |

### 2.2 音效系统 🔔

| 功能 | 描述 | 技术方案 |
|------|------|----------|
| **场景 BGM** | 低音量循环的寺庙环境音（鸟鸣、风铃声） | Web Audio API + 短音频循环（~10s），音量可调 |
| **交互音效** | 抽屉悬停→木琴轻响；捏合确认→铜锣声；换批→木鱼声 | `Howler.js` 或原生 `Audio` 缓冲池，预加载 |
| **揭签音效** | 签文展开时纸张展开的沙沙声 | 短音频 + gain 衰减 |
| **音效控制** | 页面角落的音量滑块／喇叭图标开关 | localStorage 存储偏好 |

### 2.3 签文展示增强 📜

| 功能 | 描述 |
|------|------|
| **签纸展开动画** | 抽到签后签纸从中间向外卷轴式展开，不是整个卡片闪现 |
| **签纸风格** | 签纸背景模拟和纸纹理（米黄色 + 纤维纹理），签文竖排从右到左书写 |
| **汉数字/日文编号** | 签号显示为日式汉数字「第百番」或传统中文大写 |
| **签文朗读** | 点击朗读按钮，用 TTS 或预录音频朗读签诗（中文/日文） |
| **签文注解折叠** | "愿望""疾病""失物" 等分类默认折叠，用户展开自己想看的项 |

## 3. 阶段二：交互深化 · 更丰富的仪式感

### 3.1 抽签流程再造

当前流程：选择方式 → 九宫格抽屉 → 点/捏 → 出结果

**新流程：**

```
首页（氛围展示 + 简介）
  │
  ├─ 抽签准备（可选：摇签筒）
  │   ├── 虚拟签筒出现（3D 或 Canvas 绘制的签筒）
  │   ├── 用户摇一摇（手势：摇晃手机 / 鼠标左右拖动）
  │   ├── 一支签从筒中飞出，掉落在桌面上
  │   └── 自动翻转显示编号，引导到九宫格
  │
  ├─ 传统模式（保持九宫格，但增加动画）
  │   ├── 抽屉从底部滑入（交错延迟，像抽屉依次弹出）
  │   ├── 选中后抽屉缓缓拉开
  │   ├── 签纸从抽屉中升起展开
  │   └── 签名显现（字迹渐显效果）
  │
  └─ 结果页（文字 + 图像 + 符号动画）
      ├── 签文文字一行一行渐显（打字机效果配合墨迹渗开）
      ├── 吉凶等级以大号书法字展示 + 背景色渐变变化
      ├── 下方"详细解读"以折叠卡片形式
      └── 可分享 / 收藏
```

### 3.2 手势模式增强

| 功能 | 描述 |
|------|------|
| **摇签筒手势** | 手掌握拳晃动 → 触发签筒摇晃动画 → 一支签飞出 |
| **双手模式** | 双手合十（祈祷手势）→ 自动进入抽签状态，更有仪式感 |
| **鞠躬检测** | 手部位置向下移动 → 检测到鞠躬 → 触发"尊敬的参拜"反馈 |
| **手掌展开** | 手掌朝向摄像头展开 → 签文在手掌上浮现（AR 感） |
| **手势提示** | 在 camera 预览上叠加手部骨架 + 指尖距离数值 / 手势图标提示 |

### 3.3 摇签筒动画（核心新交互）

实现一个 Canvas 签筒：
- 签筒为梯形/桶形（SVG / Canvas 绘制），里面插着数十根签棒
- **鼠标/触摸拖拽**：左右拖动签筒 → 签棒晃动（弹簧物理模拟）
- **摇动力量累积**：晃动幅度越大 → 签棒越猛烈摇晃 → 最终一根飞出
- **签棒飞出动画**：抛物线贝塞尔曲线，落在场景中翻转显示编号
- **落签音效**：竹签落在木盘上的清脆碰撞声

物理模拟方案：
- `requestAnimationFrame` 驱动的轻量弹簧模型
- 签棒用数组表示位置偏移量，晃动时叠加正弦波 + 阻尼
- 无需第三方物理引擎，手写 50 行即可

### 3.4 添加 Multiple Fortune Types

| 签种 | 交互方式 |
|------|----------|
| **综合运势（当前）** | 随机抽 1-100 号签 |
| **爱情签** | 从 100 签中按"爱情"相关度加权抽取（前端预处理 tags） |
| **事业签** | 同上 |
| **今日一言** | 抽取一句短签，适合每日快速使用 |
| **数字签** | 手写或选择一个 1-100 数字，直接看对应编号的签（保留日式"指定抽签"文化） |

## 4. 阶段三：数据进阶 · 签运统计与收藏

### 4.1 历史与统计面板 📊

| 功能 | 描述 |
|------|------|
| **扩展历史** | 从 5 条扩展到 50+ 条，新增分页/滚动加载 |
| **统计看板** | 饼图/柱状图展示吉凶分布（使用 Chart.js 微图或纯 CSS 条） |
| **运势趋势** | 折线图显示近期运势走向（按日期/抽签次数轴） |
| **运势热力图** | 按时间段（早/中/晚）展示运势分布——你什么时段运势最好？ |
| **吉凶日历** | 用颜色标注每天抽到的吉凶等级（直接感知月度运势） |
| **签文收藏** | 用户可"收藏"某支签，收藏签单独列表，随时回看 |
| **数据导入导出** | 导出 JSON / CSV 格式的抽签记录，可备份或分享给数据分析师朋友 😄 |

### 4.2 Gamification 🏆

| 功能 | 描述 |
|------|------|
| **成就系统** | 抽满 10 签/50 签/100 签 / 连续 7 天大吉 / 收集所有 100 签 / 解锁所有签种 |
| **签集图鉴** | 全 100 签图鉴页面，展示已收集/未收集 + 各签详细信息（类似宝可梦图鉴） |
| **每日签到** | 每日首次抽签有特殊提示 + 累计签到徽章 |
| **幸运周报** | 每周运势总结报告（抽到最多的吉凶等级 + 最幸运的一天） |

### 4.3 本地存储升级

现在只用 `localStorage` 存 5 条历史记录。升级方案：

```
IndexedDB (idb library)
  ├── omikuji_records 表 — 每次抽签记录 (id, fortune_id, grade, timestamp, sign_type, tags)
  ├── collections 表 — 用户收藏的签文 ID
  ├── achievements 表 — 解锁的成就列表
  ├── stats_cache 表 — 统计缓存（避免每次都全量计算）
  └── user_prefs 表 — 用户偏好（音量、主题、手势灵敏度等）
```

引入 `idb`（IndexedDB 的 Promise 封装）或直接用 `idb-keyval` 轻量方案。

### 6.2 推荐迁移路径

**当前 → 短期（不拆框架，只在现有上增量）**

```
1. main.ts 保持不变，逐步拆出模块
2. 新增 services/ 目录抽取逻辑
3. 新增 components/ 目录抽取 DOM 生成函数
```

**中期（如果感觉维护吃力）→ 引入 Svelte 或 Preact**

Svelte 优势：
- 编译型，打包极小（适合这个单页应用）
- 响应式天然适合状态管理
- 写动画比 React 更方便（`transition`, `animate`, `fly`, `fade` 开箱即用）
- 无运行时，包体积 10KB 以内

Preact 优势：
- 和当前 Vanilla JS 风格接近，迁移成本低
- 生态兼容 React

**推荐：如果当前代码量还能接受，先用模块化拆分保持 Vanilla JS。如果感觉状态管理爆炸了，果断上 Svelte。**

### 6.3 PWA 升级

| 功能 | 描述 |
|------|------|
| **Service Worker** | 缓存所有静态资源 + 签文 JSON，离线可用 |
| **manifest.json** | 添加应用名称、图标、主题色，支持"添加到主屏幕" |
| **离线策略** | 签文数据预缓存，抽签逻辑纯客户端，完全离线可用 |
| **推送通知** | 可选：每日签到提醒、运势更新提示 |

### 6.4 性能优化

| 优化项 | 方案 |
|--------|------|
| 图片加载 | `<img loading="lazy">`（已有） + 预加载首屏关键图片 + WebP 格式 |
| 签文 JSON | 按需分页加载（当前 100 条一次性加载，影响不大但可优化） |
| 手势识别 | 当前每帧都在 `querySelectorAll`，改为缓存引用 + 增量更新 |
| 粒子系统 | 使用 OffscreenCanvas（Web Worker 中运行粒子计算） |
| CSS 动画 | 使用 `will-change` + `transform` 避免重排 |
| 音效 | 预创建 Audio 缓冲池，避免每次创建新 Audio 对象 |

## 7. 技术可行性评估

| 功能 | 难度 | 风险 | 备注 |
|------|------|------|------|
| 樱花飘落 | ⭐ | 低 | Canvas 粒子，已有成熟方案 |
| 音效系统 | ⭐⭐ | 低 | 音频文件需自己准备或找免费音效 |
| 签纸展开 | ⭐⭐ | 低 | CSS `clip-path`/`mask` 动画或 Canvas 绘制 |
| 签筒物理 | ⭐⭐⭐ | 中 | 弹簧物理模拟+Canvas 绘制，不含 3D |
| 手势增强 | ⭐⭐ | 中 | 基于现有 MediaPipe 代码扩展 |
| 统计面板 | ⭐⭐ | 低 | IndexedDB + Chart.js 或纯 CSS |
| 分享图片 | ⭐⭐⭐ | 中 | html2canvas 渲染质量需调试 |
| 图鉴系统 | ⭐⭐ | 低 | 纯数据展示，代码量较大但无技术难点 |
| PWA | ⭐⭐ | 低 | Vite 有 PWA 插件 `vite-plugin-pwa` |
| Svelte 迁移 | ⭐⭐⭐⭐ | 高 | 重构较大，建议只在确实需要时做 |
